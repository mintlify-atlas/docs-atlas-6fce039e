## Overview

The JKPSC Answer Evaluator is built with modern web technologies, prioritizing client-side performance, type safety, and developer experience.

## PDF & Document Processing

### PDF Extraction

```json package.json
{
  "pdfjs-dist": "^5.4.624"
}
```

**pdfjs-dist** - In-browser PDF parsing (Mozilla's PDF.js library)

- Runs in Web Worker for non-blocking extraction
- Extracts text page-by-page
- Configured via `import.meta.url` pattern

```typescript
// src/components/PdfUpload.tsx
import * as pdfjsLib from 'pdfjs-dist';

pdfjsLib.GlobalWorkerOptions.workerSrc = new URL(
  'pdfjs-dist/build/pdf.worker.mjs',
  import.meta.url
).toString();
```

### PDF Generation

```json package.json
{
  "@react-pdf/renderer": "included via @json-render/react-pdf",
  "@json-render/core": "^0.11.0",
  "@json-render/react-pdf": "^0.11.0"
}
```

**Lazy-loaded** - Only imported when user clicks "Download PDF Report"

```typescript
// Lazy import in Index.tsx
const { downloadPdf } = await import('@/lib/pdfGenerator');
await downloadPdf({ result, subject, marks, question, answer });
```

**PDF Report includes:**
- Header with logo (base64 embedded)
- Score card with color-coded scores
- Category breakdown table
- Key points analysis (covered ✓ / missing ✗)
- Improvement tips (numbered list)
- Model answer structure
- Footer with branding

## AI & API Integration

### Google Gemini API

**Direct REST API calls** (no SDK) to:

```
https://generativelanguage.googleapis.com/v1beta/models/{model}:generateContent
```

**Models used:**

| Use Case | Model | Config |
|----------|-------|--------|
| **Answer evaluation** | `gemini-3-flash-preview` | temp: 0.3, maxTokens: 65536 |
| **Image transcription** | `gemini-2.5-flash` | Vision API for handwriting OCR |

**Key features:**
- No backend proxy - API key stored in localStorage
- Web Worker execution keeps UI responsive
- Fallback to main thread if Worker unavailable
- 3-stage JSON repair (direct parse → auto-repair → regex extraction)

## Data Management

### IndexedDB

```json package.json
{
  "idb": "^8.0.3"
}
```

**idb** - Promise-based IndexedDB wrapper by Jake Archibald

```typescript
// src/lib/historyDb.ts
import { openDB, DBSchema } from 'idb';

interface JKPSCPrepDB extends DBSchema {
  evaluations: {
    key: string;
    value: HistoryEntry;
    indexes: {
      'by_timestamp': number;
      'by_subject': string;
    };
  };
}

const db = await openDB<JKPSCPrepDB>('jkpscprep', 1, { ... });
```

**Exported functions:**
- `getAllEntries()` - Fetch all eval history
- `putEntry(entry)` - Save new evaluation
- `deleteEntryById(id)` - Remove single entry
- `clearAllEntries()` - Wipe all history
- `getFilteredEntries(filter)` - Subject/date filtering
- `countEntries()` - Total count

<Info>
The old `jkpscprep_eval_history` localStorage key is auto-migrated to IndexedDB on first load.
</Info>
